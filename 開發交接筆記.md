# CYT YouTube 下載器 v3.0 開發交接筆記 (Flet 遷移版)

## 1. 專案現況概要
目前專案正處於從 **v2.1.0 (Tkinter)** 遷移至 **v3.0 (Flet)** 的關鍵階段。
主要的目標是透過 Flet 框架實現現代化、跨平台（Web/Desktop）且高效能的介面，目前核心下載功能已成功跑通。

## 2. 技術棧與環境
- **語言**: Python 3.13
- **介面框架**: Flet 0.84.0 (最新版，具有重大 API 變動)
- **核心引擎**: `yt-dlp` (解析與下載), `ffmpeg` (轉檔與合併)
- **執行方式**: `python app_flet_poc.py` (目前預設為 Web 瀏覽器模式)

## 3. 架構說明 (重要)
專案已完成 **邏輯與介面分離 (Decoupling)**：
- **`core_engine.py`**: 核心業務邏輯。
    - 封裝了 `yt-dlp` 的解析與下載邏輯。
    - 支援異步解析、進度回調 (Progress Hook)。
    - 支援「批量單一影片下載」模式，解決了播放清單會直接下載全集的限制。
- **`app_flet_poc.py`**: Flet 非同步介面。
    - 採用 **Async 模式** (`async def main`)。
    - 包含：網址解析、播放清單勾選列表、下載選項 (MP3/MP4 及音畫質切換)、即時進度條。

## 4. Flet 0.84.0 技術坑位 (接手者必讀)
此版本有許多與舊教學不相容的變更，已在程式碼中修復：
1. **全大寫常數**: 所有的 `ft.colors`、`ft.icons`、`ft.alignment` 必須改為大寫開頭，如 `ft.Colors.BLUE`。
2. **FilePicker 變革**: 
    - 不再支援 `on_result` 回調。
    - 不可使用 `page.add()` 或 `page.overlay.append()`。
    - 必須使用 `await picker.get_directory_path()` 直接獲取結果。
3. **組件參數**:
    - `Tabs` 必須提供 `length` 參數。
    - `Text` 的裝飾性參數 (如 `letter_spacing`) 必須放在 `style=ft.TextStyle(...)` 中。
    - `ElevatedButton` 建議使用 `content=ft.Text(...)` 而非 `text="..."`。
4. **Web 模式限制**: 瀏覽器不支援 `get_directory_path`，已實作「手動輸入路徑」作為備援。
